# A Hybrid Beam Search Framework with Deep Representation Learning for the Multiple Longest Common Subsequence Problem

## Overview

This project provides a framework for solving the Multiple Longest Common Subsequence (MLCS) problem using a hybrid beam search approach. By integrating deep representation learning with heuristic search, the framework leverages a lightweight Transformer encoder to capture local contextual features. This addresses the limitations of existing approximation methods in processing large-scale biological sequences, effectively balancing computational efficiency with solution quality.

The analysis workflow consists of the following stages:
1. **Data Preprocessing and Model Training**:
   - Slicing DNA sequences using a sliding window mechanism.
   - Training a lightweight Transformer encoder via contrastive learning (Triplet Loss) to learn semantic embedding vectors for sequence fragments.
2. **Hybrid Beam Search Execution**:
   - **Non-dominated Filtering**: Performing preliminary screening of search branches to eliminate invalid states.
   - **Dual-bound Evaluation**: Assessing candidate paths using an improved heuristic upper bound (theoretical potential) and a structural lower bound (alignment compactness).
   - **Multi-dimensional Scoring and Pruning**: Integrating representation-based priority with dual-bound information to retain Top-K high-potential paths.

## Dataset Description

The datasets are organized into `random-data/` and `real-data/` folders, with all sequences provided in standard `.fasta` format.

### 1. Simulated Datasets (`random-data/`)
These are generated by randomly sampling nucleotides to evaluate performance variations across different scales.
* **`random-data/length/`**: The number of sequences is fixed at 50, with lengths varying from 300 to 800.
* **`random-data/number/`**: The sequence length is fixed at 1000, with the number of sequences varying from 15 to 75.

### 2. Real Biological Datasets (`real-data/`)
Sourced from the NCBI public database, these include complete genome sequences of bacteriophages infecting opportunistic pathogens such as *Escherichia*, *Klebsiella*, and *Hafnia*.
* **`real-data/length/`**: Used to verify effectiveness under real biological constraints like uneven base distributions and local variations.
* **`real-data/number/`**: Used to test stability and robustness under high-dimensional constraints.

## How to Perform MLCS Analysis

### 1. Prepare Input Data
Prepare your sequence files in FASTA format. We recommend following the structure in the `data/` directory, where each experimental configuration includes 5 independent test groups (`group1` to `group5`) to ensure statistical reliability.

### 2. Execute the Analysis
Use the main script `main.py` to start the search process.
* **Key Parameter Descriptions**:
  - `--beam_width` (K): Search beam width. The recommended default is 15 to balance precision and efficiency.
  - `--window_size` (W): Transformer sliding window size. The recommended default is 10.
  - `--step_size` (S): Sampling step size for training. The recommended default is 40.

### 3. Output
The algorithm maintains the `Best_MLCS` in real-time and outputs the final longest common subsequence and its length upon completion.
